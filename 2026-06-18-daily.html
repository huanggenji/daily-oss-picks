今日学一招
CODEX CLI · 实战技巧

Codex CLI 非交互模式（exec）：把任务塞进脚本和 CI 自动跑

平时用 Codex 都是开个交互窗口一来一回，但很多活儿其实可以「一次性给指令、自动跑完、拿结果」——比如批量生成提交说明、定时整理日报、在 CI 里自动跑检查。Codex CLI 的非交互模式就是为此而生，让 AI 从「陪你聊」变成「替你跑」。

  • 核心是把指令直接当参数传进去一次跑完，例如 codex exec "总结这个仓库最近的改动并生成 changelog"，不进入交互界面，跑完即退出，适合写进脚本。
  • 结合管道与重定向把产物落地：codex exec "…" > out.md，或把 git diff 通过管道喂给它生成 commit message，一条命令搞定。
  • 放进 CI 时务必收紧权限：用只读/受限审批模式跑，明确允许的目录和命令，避免自动化环境里 AI 误改或误删文件。
  • 给非交互任务写清「输入是什么、要输出什么格式」，指令越具体结果越稳定，因为它没法像交互时那样反问你澄清。
  • 典型场景：定时任务里用 codex exec 生成日报/周报、PR 流水线里自动出代码摘要、批量为多个文件套同一套改写规则。
