今日学一招
CODEX CLI · 实战技巧

Codex CLI 用 profiles 管理多套配置：按项目和场景一键切换模型与审批策略

同一个人，写业务代码、改线上脚本、做实验探索，需要的「模型 + 审批严格度 + 工作目录」往往完全不同。每次手动改参数既麻烦又容易出错。Codex CLI 的 profiles 机制就是把几套常用配置预先命名存好，用一个名字一键切换，让 AI 在不同场景下既好用又安全。

  • 在配置文件 ~/.codex/config.toml 里用 [profiles.xxx] 段落定义多套配置，每套可单独指定模型、审批模式、可写目录等，互不干扰。
  • 用时只需 codex --profile work 指定 profile，或在配置里设 profile = "work" 作默认，省去每次敲一长串参数。
  • 建议按风险分档：日常开发用宽松一点的 profile，碰线上/生产相关的活儿切到只读、严格审批的 profile，让权限和场景匹配。
  • 给不同项目配不同模型也很实用：成本敏感的批量任务用便宜模型的 profile，复杂重构切到能力更强模型的 profile。
  • 典型场景：dev 快速迭代、review 只读审查、ci 非交互严格模式，三个 profile 覆盖大半日常，配置一次长期复用。
