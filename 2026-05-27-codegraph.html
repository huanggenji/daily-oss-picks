一句话定位

CodeGraph 是一个本地优先的代码智能工具：它用 tree-sitter 解析你的代码库，把所有符号、调用边、继承关系存入本地 SQLite，再通过 MCP 协议把这张"代码知识图"暴露给 Claude Code、Cursor、Codex 等 AI 助手——让 AI 直接查图谱而不是反复 grep / Read，一次索引，持续省钱省时。

它解决了什么问题

当 AI 编程助手处理一个中大型项目时，大量时间与 Token 都花在发现阶段——反复调用 grep、glob、Read 来搞清楚代码是怎么组织的。典型场景是：

  • 「这个函数被谁调用了？」→ AI 要扫 N 个文件
  • 「修改这个类会影响哪些下游模块？」→ AI 要从头追溯依赖链
  • 「这个接口在哪里实现？」→ AI 要逐文件全文搜索

CodeGraph 把这一切"提前做好"：一次性把整个代码库的结构关系索引到本地 SQLite，后续 AI 查一次图谱即可得到精确答案，无需再读文件。

核心架构（架构师视角重点）

整体流水线

源码文件 → Extraction（tree-sitter AST 解析） → Storage（SQLite / FTS5） ↓ Resolution（引用解析 + 框架模式识别） ↓ Graph Queries（callers / callees / impact radius） ↓ Context Building（Markdown / JSON → AI 消费）

1. Extraction：AST 驱动，绝不靠 LLM 猜

CodeGraph 用 tree-sitter（多语言 AST 解析器）提取符号，而非用 LLM 总结代码。这意味着抽取结果是确定性的（同一代码永远产出相同图谱）、高速的（纯本地计算），且不消耗任何 Token。支持 20+ 语言，包括 Java、TypeScript、Python、Go、Rust 等。

2. Storage：SQLite + FTS5 的极简选择

所有节点（函数、类、方法、类型）与边（调用、导入、继承、实现）都存入本地 .codegraph/codegraph.db，使用 SQLite 的 FTS5 全文索引模块加速模糊查找。这个选型非常务实：无需独立数据库进程，无运维负担，单文件即可备份和迁移，对只读查询有很好的性能。

3. Resolution：处理动态特性与框架惯例

纯 AST 解析只能看到静态结构，无法处理动态派发、回调、React 重渲染等模式。CodeGraph 在 Resolution 阶段额外引入了框架合成器（Synthesizers），专门识别 Spring / Express / Next.js 等框架的路由注解、依赖注入等惯例，把这些"隐式边"补入图谱，让调用链从头到尾连通。

4. MCP Server + Auto-sync：增量、实时

CodeGraph 作为一个 MCP Server 运行，常驻后台，通过系统原生文件事件（FSEvents / inotify）监听代码变更，增量重索引变动文件，让图谱始终与代码同步。AI 助手通过标准 MCP 协议查询，不需要任何特殊配置。

5. 多 Agent 集成架构

CodeGraph 提供统一的 MCP 接口，同一个本地图谱可以同时服务 Claude Code、Cursor、Codex CLI、opencode、Hermes Agent。安装时通过 targets/ 插件体系自动写入对应 Agent 的配置文件，做到一次安装多端共享。

架构亮点：这是一个典型的"索引换查询"（Index-for-Query）设计模式——通过提前构建结构化索引，把运行时的 O(n) 扫描转化为 O(1) 图查询，本质和数据库索引、搜索引擎倒排索引、Elasticsearch 是同一套思想。

适用场景

  • 大型单体 / 微服务项目：代码库越大，AI 扫文件的开销越高，CodeGraph 节省越明显。对 Java 微服务项目（几十个模块、数千个类）尤其适合。
  • AI 辅助重构：在重命名、提取接口、调整包结构前，让 AI 先查 Impact Analysis（影响半径），评估改动波及哪些调用链。
  • 新人 Onboarding：用 AI + CodeGraph 快速建立对陌生大型代码库的结构认知，替代人工阅读文档。
  • CI 中的精准测试选择：通过 impact 图谱只跑被改动代码影响到的测试集，大幅减少 CI 时间（官方文档有专门 Guide）。
  • 代码审查辅助：AI 用图谱快速追溯 PR 改动的上下游影响范围。

给你的学习要点

🏗️ 可迁移的架构思想

  • 索引优先（Index-first）设计：任何需要"频繁查询大量数据"的场景，都应该先问自己"能不能提前建索引？"。CodeGraph 把 AI 的 Runtime 发现成本转移到 Build-time 索引成本，这是系统设计中「空间换时间」思想的经典应用。Java 生态里的 Spring Boot 启动时扫描 Bean、MyBatis 启动时解析 Mapper XML，都是同一套思路。
  • 确定性胜过智能：提取符号用 tree-sitter AST，而非 LLM 总结——这是一个刻意的架构决策。确定性意味着可测试、可 diff、可信赖。在 AI 时代，越是核心的基础设施层，越要保持确定性，用 AI 做高层推理而非低层解析。
  • 协议标准化带来的生态红利：CodeGraph 通过支持 MCP 协议，一次性接入了所有支持 MCP 的 AI 助手。这和 JDBC、OpenAPI、gRPC 的逻辑是一样的——定义标准接口，生态自然汇聚。
  • 轻量存储选型：选 SQLite 而非 PostgreSQL 或 ElasticSearch，是因为它的使用场景就是"单机、低并发、读多写少"——选型与场景匹配，才是好的工程决策，而不是总用最重的武器。
