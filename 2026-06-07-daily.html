精选 01 · AI Agent / 生成式 UI

CopilotKit/CopilotKit

TypeScript⭐ 32.8kMIT

面向 AI Agent 的前端框架，也是 AG-UI 协议的发起方。它让 Agent 不再只会输出大段文字，而是能直接「画」出界面——实时图表、可交互组件、人工审批环节、前后端共享状态都能由 Agent 动态生成。

为什么值得看：Agent 的下一步是「会用 UI 行动」而不是只会聊天。CopilotKit 把生成式 UI 的脚手架（useAgent、useFrontendTool、Headless UI）打包好，兼容 LangGraph、Mastra、MCP 等主流栈，是把 Agent 接进真实产品界面的最短路径。

📚 今日 AI 学习
Codex 系列

给 Codex CLI 写自定义 Skill / Prompt，把重复活儿一键化

前几天我们讲了用 AGENTS.md 让 Codex 懂你的代码库、用 MCP 接外部工具。今天补上「自定义 Skill / Prompt」这一块——它解决的是：那些你每天都要让 Codex 做、且做法固定的事（写 commit、生成测试、补 docstring），如何沉淀成一条命令复用。

1. 什么是 Codex 的自定义 Prompt

自定义 Prompt 本质是一段「预写好的指令模板」，你把它存成文件，之后用一个短名字就能反复调用，省得每次都把要求打一遍。它是把「工作流」固化下来的最轻量方式。

2. 创建一个：自动写规范 commit message

# 在项目里新建 prompts 目录并写一个模板
mkdir -p .codex/prompts
cat > .codex/prompts/commit.md << 'EOF'
读取当前 git diff（已 staged 的改动），
生成一条符合 Conventional Commits 规范的提交信息：
- 首行格式 type(scope): 简短描述，不超过 50 字
- 空一行后用 3~5 条要点说明改了什么、为什么
- type 从 feat/fix/refactor/docs/test/chore 中选
只输出 commit message 本身，不要多余解释。
EOF

3. 调用它

# 先 stage 你的改动
git add -A

# 让 Codex 按模板生成（具体命令以你本地 codex 版本的 prompt 调用语法为准）
codex "按 .codex/prompts/commit.md 的要求，基于当前 staged diff 生成 commit message"

4. 进阶：把它升级成可复用 Skill

当一个 Prompt 需要多步操作（读 diff → 生成 → 自动执行 git commit）时，就该升级成 Skill。Skill 比 Prompt 多了「能调工具、能分步执行」的能力，相当于给 Codex 装一个小型自动化脚本，触发词一喊就跑完整条流水线。

💡 一句话记住：Prompt 固化「怎么说」，Skill 固化「怎么做」。把每天重复 3 次以上的 Codex 操作写成 Prompt/Skill，是个人效率提升最高性价比的一步。
