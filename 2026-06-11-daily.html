🔥 今日开源精选
mvanhorn/last30days-skill ⭐ 32.2k Python AI Agent · 全网情报研究
一个让 AI 跨平台「全网调研」的 Agent 技能，自动汇总最近 30 天的真实讨论

普通搜索引擎前几页常被 SEO 软文和 AI 洗稿占满，真正有价值的观点往往埋在 Reddit 评论、X 帖子或 Hacker News 讨论里——而这些来源没有任何一个搜索框能一次搜全。last30days-skill 把这件事变成一条命令。

它是一个 AI Agent 技能：给定一个主题，它会并行抓取 Reddit、X、YouTube、Hacker News、Polymarket、GitHub 等多个平台最近 30 天的内容，按真实热度打分排序，再合成一份带引用来源的总结报告，还能直接输出 HTML 简报。可配合 Claude Code、Codex、Cursor 等任何能跑 shell 命令的助手使用。

适合谁：做市场调研、追技术热点、想快速摸清一个新项目口碑的人。把"到处翻评论区"的考古活儿，交给一个不睡觉的研究助手。
safishamsi/graphify ⭐ 63.5k Python AI 编程 · 代码知识图谱
一条命令，把整个项目（代码 + 文档 + 截图 + 论文）变成 AI 可查询的知识图谱

用 AI 助手理解大项目时有个反复出现的痛点：每次提问，它都要重新读文件、找线索、拼上下文。代码在 src、文档在 docs、设计说明和截图散在别处，资料很多但彼此的关系从没沉淀下来。

graphify 是给 AI 编程助手用的技能：跑一次 /graphify，它就读一遍整个仓库——不只是代码，还有 SQL schema、shell 脚本、Markdown、PDF、论文甚至截图——抽出其中的概念和它们的连接关系，生成 graph.html、报告和 graph.json。之后 AI 先查这张关系图，再决定该读哪段上下文，少读文件、少绕弯路。支持 Claude Code、Codex、OpenCode、Cursor、Gemini CLI 等。

适合谁：在大型或多语言仓库里用 AI 写代码的人。把"项目结构"一次性沉淀成图，让助手每次都从地图出发而不是从零开始。
TapXWorld/ChinaTextbook ⭐ 74k Roff 教育资源 · 中文教材
把小学到大学的中文 PDF 教材集中整理开源，自学和备课的连续资料入口

这是一个把中国小学、初中、高中到大学各阶段 PDF 教材集中整理的开源项目，目标是为基础薄弱地区和海外中文家庭提供可连续获取的教育资源。资料按学段组织，查找和下载都更集中。

它的价值不在技术，而在"系统性"：自学者能低成本拿到一整套连贯的教材补基础，家长能给孩子找到对应学段的系统资料，海外中文学习家庭也有了清晰的资料入口。和今天另两个 AI 工具放在一起，它提醒我们开源也可以是把公共知识更公平地铺开。

适合谁：自学补基础、给孩子找系统教材、或做中文教学资料整理的人。一个把"散落的公共教育资源"集中起来的高星项目。
📚 今日一学：Codex CLI 用 @ 精准喂上下文
Codex 系列 · 上下文管理

用 @ 引用文件，让 Codex 看你想让它看的那部分

很多人用 Codex CLI 时习惯把需求一句话丢过去，然后抱怨它"没看懂代码"。问题往往不在模型，而在于你没告诉它该看哪里。Codex CLI 支持用 @ 直接在对话里引用文件或目录，把精确的上下文塞进这一轮请求——这是省 token、提准确率最实用的一招。

在交互式会话里，输入 @ 会触发文件路径补全。你可以引用单个文件，也可以引用整个目录：

> 解释一下 @src/auth/login.ts 的登录流程，
  有没有可以简化的地方？

> 参照 @src/components/Button.tsx 的写法，
  帮我给 @src/components/ 里缺测试的组件补单测

这样做的好处很直接：Codex 只会把你点名的文件读进上下文，而不是把半个仓库塞进去。文件越精准，回答越贴题，token 也越省。当你只想让它改某一处时，引用那一个文件，比让它"自己去找"可靠得多。

💡 实战习惯：改动前先用 @ 把相关文件和参照样例都点名给它（"照着 A 文件的风格改 B 文件"），再描述需求。给足精确上下文，往往一轮就能拿到能用的结果，而不是来回纠正三四次。
