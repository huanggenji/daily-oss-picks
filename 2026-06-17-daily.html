Trendshift #18

hugohe3/ppt-master

一句话定位：把任意文档喂进去，AI 生成的是带原生形状和动画、能在 PowerPoint 里直接改的真 PPTX，而不是一堆图片堆出来的假幻灯片。

⭐ 20,600+ Python Trendshift #18
  • 多数 AI 生成 PPT 工具给的是「图片版」幻灯片，改不动；ppt-master 输出的是原生文本框、图表、形状，回到 PowerPoint 可逐字编辑。
  • 支持原生动画、演讲者备注，甚至能按你自己的 .pptx 模板出片，保持品牌与版式一致。
  • Python + MIT 协议，适合接进自己的内容流水线，把「文档 → 可交付 PPT」这一步自动化。

Trendshift #12

trycua/cua

一句话定位：开源的 Computer-Use Agent 基础设施，给 AI 一整台带沙箱的电脑去操作，可在 macOS / Linux / Windows 上控制完整桌面。

⭐ 18,400+ HTML Trendshift #12
  • 提供沙箱、SDK 与基准测试，让 AI Agent 能像人一样点鼠标、敲键盘、管理窗口、操作文件，覆盖三大桌面系统。
  • 支持后台运行：用虚拟鼠标在后台同时操作多个程序，不抢占你正在用的真鼠标，多 Agent 各自绑定一个窗口互不打架。
  • 相比只能在浏览器里跑的方案，cua 给的是完整桌面控制权，适合把 Windows-only 的老业务系统也纳入自动化。

CODEX CLI · 实战技巧

Codex CLI 会话恢复与历史管理：关掉终端也能接着干

长任务做到一半被打断、电脑要重启、或者第二天想接着上次的上下文继续？Codex CLI 把每次对话都存成可恢复的会话，不必从头再喂一遍背景。学会恢复历史会话，能省下大量重新解释项目的 token，也让多任务并行更从容。

  • 新开会话正常运行 codex 即可；任务做到一半退出也没关系，会话历史会自动保存在本地。
  • 想接着上一次继续，用 codex resume 列出最近的会话并选择恢复，原来的上下文、文件引用和对话都还在。
  • 只想快速接上「最后一次」对话，用 codex resume --last 一步回到上次现场，适合临时被打断后立刻续上。
  • 多任务时给会话起清晰主题、定期用 /compact 压缩上下文，恢复后既不爆 token，又能让 AI 记得关键决策。
  • 小技巧：把恢复会话和 git 分支对应起来——一个分支一条会话，切分支时 codex resume 选对应会话，思路不串味。
