一句话定位：一份描述视觉风格的格式规范，让编程 Agent 持续、结构化地理解你的设计系统。
⭐ 17,400+|TypeScript·Trendshift #1

📘 今日学一招

用 Codex CLI 打通从 issue 到 PR 的完整流程

很多人用 Codex 只让它改一处代码，其实它能把「读需求 → 拉分支 → 改代码 → 写 PR 描述」串成一条流水线。关键是给够上下文、小步验证、让它产出可直接提交的成果。
  • 先用 codex 进入交互模式，把 issue 正文贴进去或用 @ 引用相关文件，让它先复述需求并列出改动计划，确认无误再动手。
  • 让它在新分支上工作：先 git checkout -b feat/xxx，每完成一小步就用 /diff 看清改动、跑测试，再让 Codex 生成符合规范的 commit message。
  • 收尾时让 Codex 读完整 diff（git diff main）后写 PR 标题与描述：动机、改了什么、如何验证；自己审一遍再 gh pr create 提交，AI 出草稿、你把关。
