📘 今日学一招

提示词工程入门：把模糊指令写成大模型能精准执行的提示

同样一个需求，提示写得好不好，模型输出质量能差出一大截。提示词工程不是玄学，核心就是把「你脑子里默认知道、但没说出口」的信息补全，让模型不用猜。掌握几条基本结构，就能稳定拿到更靠谱的结果。
  • 讲清角色 + 任务 + 约束 + 输出格式：与其说「帮我写个函数」，不如说「你是资深 Python 工程师，写一个校验邮箱的函数，用标准库、加类型注解，只返回代码」。
  • 给示例胜过讲道理：想要固定格式时，直接给一两个「输入→输出」样例（few-shot），比用文字描述格式更不容易跑偏。
  • 让它先想再答、复杂任务拆步骤：加一句「先列出思路，再给结论」往往更准；同样的技巧用在 codex 或 Claude Code 上，把需求写细、给上下文，AI 写代码也会少返工。
