① Trendshift #5

simplex-chat/simplex-chat

一句话定位：第一个完全不依赖任何用户标识的即时通讯网络，从设计上做到 100% 私密。
⭐ 12,400+|Haskell·Trendshift #5
  • 没有用户 ID、电话号、用户名甚至随机密钥作为身份标识——服务器看不到「谁在和谁聊天」，从根上消除元数据泄露。
  • 采用一次性连接队列与双棘轮端到端加密，消息内容和社交图谱都不会被中转服务器掌握。
  • iOS、Android、桌面端全平台开源，可自建中继服务器，是注重隐私的去中心化通讯代表项目。
② Trendshift #13

IceWhaleTech/CasaOS

一句话定位：简单优雅的开源个人云系统，把闲置的旧电脑或 NAS 变成自己的私有云。
⭐ 35,500+|Go·Trendshift #13
  • 以图形化「桌面」管理 Docker 应用，点几下就能装好 NAS、相册、影音、下载器等自托管服务，几乎零运维门槛。
  • 内置应用商店与文件管理、磁盘管理，把零散的家庭服务器需求收拢进一个统一、好看的界面里。
  • Apache 2.0 协议，曾多次登顶 GitHub Trending，是自托管「个人云」入门最受欢迎的选择之一。
③ Trendshift #10

NanmiCoder/MediaCrawler

一句话定位：覆盖小红书、抖音、快手、B 站、微博、贴吧、知乎的多平台内容与评论爬虫。
⭐ 52,600+|Python·Trendshift #10
  • 基于 Playwright 无头浏览器抓取，复用登录态拿到真实页面数据，省去逆向签名算法的繁琐过程。
  • 统一了多个平台的笔记/视频/帖子与评论采集接口，是做舆情分析、内容研究、数据集构建的常用上游工具。
  • 请遵守各平台服务条款与法律法规，仅用于学习研究与合规场景，注意控制频率、尊重 robots 与隐私边界。
📘 今日学一招

让 Codex CLI 联网搜索，写代码前先查最新文档

模型的知识有截止日期，遇到新版本库、刚改的 API，凭记忆写很容易出错。给 Codex 开上联网搜索，让它先查最新官方文档和示例，再动手写代码，能明显减少「一本正经地写错」的情况。
  • 在交互模式里用带搜索能力的命令启动（如 codex --search），让 Codex 在不确定时主动去查官方文档而不是硬编造。
  • 提问时把意图说清楚：「先查一下 X 库最新版的写法，再据此实现」，并给出版本号，引导它以检索到的内容为准。
  • 拿到结果后让它附上来源链接，自己抽查一两处再采纳；联网只补「最新事实」，代码逻辑仍要靠 /diff 和测试把关。
