用 Codex CLI 做代码审查：让 AI 先帮你 review 再提 PR

写代码靠 AI，审代码同样可以靠 AI。Codex CLI 能直接读取 git diff，从可读性、潜在 Bug、边界条件、安全风险几个维度给出逐行点评——相当于在提交 PR 之前先过一轮"机器同事评审"。

  • 审本地改动：暂存或不暂存都行，先看 diff 再让它评审：git diff | codex "扮演资深评审，逐行指出潜在 Bug、边界条件与可读性问题"
  • 审某个范围的提交：git diff main...HEAD | codex "总结这批改动的风险点，按严重程度排序"，快速摸清一个分支整体质量。
  • 聚焦安全：给它一个明确视角，例如"只找注入、越权、空指针、并发安全问题"，输出会比泛泛而谈精准得多。
  • 让它给修改建议：评审完追问一句"针对最严重的两个问题，给出修改后的代码"，直接拿到可落地的 patch。
小技巧：把常用的评审提示词写成一个自定义 Skill / Prompt（前几期讲过），以后一句 codex /review 就能跑完整套审查流程，团队风格也能统一。
