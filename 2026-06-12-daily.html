📘 今日 AI 学习
CODEX 系列

用 Codex CLI 做多文件批量重构：一次改对一整片代码

重构最痛的不是改单个函数，而是「改一个名字要同步改十几个文件」。手工改容易漏、容易引入笔误。Codex CLI 擅长这类跨文件的机械改动——你描述意图，它读懂调用关系，统一改完再让你审。

1. 先把范围喂清楚

用 @ 把相关目录或文件交给 Codex，让它先建立全局认识，而不是盲改：

codex "把 utils/ 下的 getUserData 重命名为
fetchUserProfile，同步更新所有调用处" @src/utils @src/api @src/pages

明确给出目录范围，比让它自己全库搜索更省 token、更不容易误伤同名但无关的代码。

2. 让它先列计划再动手

大范围改动前，先要一份「会改哪些文件、改什么」的清单：

codex "先别改代码，列出这次重命名会影响的
所有文件和具体改动点，我确认后再执行"

这一步能把「漏改」和「过度改」在执行前就拦下来，尤其适合涉及十几个文件的重构。

3. 小步提交，配合 git 验证

每完成一类改动就让它跑测试 / 编译，再生成 commit：

codex "重构完成后运行测试，全部通过再帮我
写一条 commit message，按 conventional commits 格式"

把大重构拆成「改 → 验 → 提交」的小循环，出问题时 git diff 一眼就能定位，回退也只丢一小步。

💡 一句话记住：范围讲清楚 → 先看计划 → 小步验证提交。机械的批量改动交给 Codex，你只负责审关键决策。
