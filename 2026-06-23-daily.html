今日学一招
CODEX CLI · 实战技巧

用 Codex CLI 的 /diff 在动手前看清 AI 到底要改什么

让 AI 改代码最怕的不是它改不动，而是它改了你没看见。Codex CLI 提供了查看变更差异的能力，在把改动落到磁盘、提交 git 之前，先把「AI 想动哪些文件、改了哪几行」摊开给你看。养成「先 diff 再确认」的习惯，能在出问题前一步拦住跑偏。

  • 在会话里用 /diff 查看当前这轮 AI 产生的改动：哪些文件被新增/修改/删除、具体改了哪几行，一目了然。
  • 把它当成「落盘前的最后一道关」：先 /diff 审一遍，确认无误再批准写入，避免误改无关文件或埋下隐藏 bug。
  • 配合审批模式使用——审批模式负责「要不要让它动手」，/diff 负责「动手前先看清动了什么」，一前一后形成双保险。
  • diff 看着不对就别急着接受：直接追加一句自然语言让 AI 修正，比改完落盘再用 git 回滚要省事得多。
  • 改动确认后再交给 git：先 /diff 心里有数，再让 Codex 帮你写 commit message，提交记录也更干净可追溯。
